# Recruiter Evidence Index

This index turns the portfolio into a fast validation path for recruiters, hiring managers and technical interviewers.

All materials are fictionalized, defensive and sanitized. They demonstrate work structure, judgment and communication without exposing internal company information.

## 10-Minute Review Path

| Step | File | What to validate |
| --- | --- | --- |
| 1 | `README.md` | Positioning, target roles and seniority signals |
| 2 | `PROJECTS.md` | Project map by hiring track |
| 3 | `projects/security-by-design-framework/README.md` | Architecture, risk and control mapping |
| 4 | `projects/secure-azure-landing-zone/README.md` | Cloud security, IAM, governance and monitoring |
| 5 | `projects/devsecops-api-security-pipeline/README.md` | Secure SDLC, AppSec, API security and quality gates |
| 6 | `projects/ai-governance-secure-rag/README.md` | AI governance, secure RAG and responsible AI controls |
| 7 | `SKILLS-EVIDENCE-MATRIX.md` | Skill-to-evidence mapping for the target role |
| 8 | `CASE-STUDIES.md` | Senior-level narratives and decision making |

## Public Safety Boundary

The artifacts use fictional names, synthetic examples and generic architecture patterns. They do not contain credentials, tokens, real IP addresses, exploit instructions, internal architecture, private operational data or confidential employer information.
